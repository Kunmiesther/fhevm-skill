# 03 — FHE Operations

## Arithmetic

```solidity
euint64 sum  = FHE.add(_a, _b);      // encrypted + encrypted
euint64 sum  = FHE.add(_a, FHE.asEuint64(10)); // encrypted + plaintext
euint64 diff = FHE.sub(_a, _b);
euint64 prod = FHE.mul(_a, _b);
euint64 quot = FHE.div(_a, 5);       // ⚠️ divisor must be PLAINTEXT uint
euint64 rem  = FHE.rem(_a, 5);       // ⚠️ same — plaintext only
euint64 mn   = FHE.min(_a, _b);
euint64 mx   = FHE.max(_a, _b);
euint64 neg  = FHE.neg(_a);          // two's complement negation
```

> **`div` and `rem` only work with plaintext (unencrypted) divisors.** Encrypted
> divisors are not supported. Design your contract logic accordingly.

## Bitwise

```solidity
euint64 r = FHE.and(_a, _b);
euint64 r = FHE.or(_a, _b);
euint64 r = FHE.xor(_a, _b);
euint64 r = FHE.not(_a);
euint64 r = FHE.shl(_a, 3);   // shift by plaintext amount
euint64 r = FHE.shr(_a, 3);
euint64 r = FHE.rotl(_a, 3);
euint64 r = FHE.rotr(_a, 3);
```

## Comparison (return `ebool`)

```solidity
ebool eq  = FHE.eq(_a, _b);     // equal
ebool ne  = FHE.ne(_a, _b);     // not equal
ebool lt  = FHE.lt(_a, _b);     // less than
ebool le  = FHE.le(_a, _b);     // less than or equal
ebool gt  = FHE.gt(_a, _b);     // greater than
ebool ge  = FHE.ge(_a, _b);     // greater than or equal
```

All return `ebool` — an encrypted boolean. You cannot branch on it with `if`.

## Conditional Logic — `FHE.select()` (CMUX)

This is the FHE equivalent of the ternary operator `condition ? a : b`.
It is the ONLY way to do conditional logic on encrypted values.

```solidity
// FHE.select(condition: ebool, ifTrue: euintX, ifFalse: euintX) → euintX
euint64 result = FHE.select(condition, valueIfTrue, valueIfFalse);
```

**Pattern: conditional transfer / guarded update**
```solidity
// Only update balance if sender has enough funds
ebool hasEnough = FHE.ge(_balances[msg.sender], amount);
_balances[msg.sender] = FHE.select(
    hasEnough,
    FHE.sub(_balances[msg.sender], amount),
    _balances[msg.sender]   // unchanged if not enough
);
_balances[recipient] = FHE.select(
    hasEnough,
    FHE.add(_balances[recipient], amount),
    _balances[recipient]
);
```

## On-Chain Random Number Generation

```solidity
euint64 rand = FHE.randEuint64();    // uniform random encrypted value
euint8  rand = FHE.randEuint8();
// Also: randEuint16, randEuint32, randEuint128, randEuint256
```

Random values are generated by the coprocessor and returned as encrypted handles.
They are not predictable by anyone, including validators.

## Chaining Operations

Operations can be chained freely — intermediate handles are resolved by coprocessors:

```solidity
euint64 fee      = FHE.div(amount, 100);          // 1%
euint64 netAmount = FHE.sub(amount, fee);
euint64 newBal    = FHE.add(_balances[to], netAmount);
ebool   isValid   = FHE.gt(newBal, _balances[to]); // overflow check
_balances[to]     = FHE.select(isValid, newBal, _balances[to]);
FHE.allowThis(_balances[to]);
```

## Cost Awareness

FHE operations are more gas-expensive than plaintext. Optimise by:
- Minimising the number of FHE operations per transaction
- Using `FHE.select()` rather than multiple conditional paths
- Batching related mutations in a single function
